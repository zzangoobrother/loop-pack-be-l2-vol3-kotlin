Tidy First: 동작 변경 없이 구조적 변경만 수행한다.

대상/방향: $ARGUMENTS

## 절차

1. 전체 테스트가 통과하는 상태인지 확인한다: `./gradlew test`
   - 실패하는 테스트가 있으면 중단하고 보고한다
2. `$ARGUMENTS`가 있으면 해당 대상의 구조적 변경을 수행한다
3. `$ARGUMENTS`가 없으면 최근 변경된 코드에서 정리가 필요한 부분을 찾아 제안한다
4. 구조적 변경을 수행한다 (아래 유형 중 선택):
   - **이름 변경**: 변수, 메서드, 클래스명을 더 명확하게
   - **메서드 추출**: 긴 메서드에서 의미 있는 단위를 분리
   - **코드 이동**: 적절한 클래스/패키지로 재배치
   - **unused import 제거**: 불필요한 import 정리
   - **중복 제거**: 반복되는 코드를 통합
   - **가드 절 도입**: 중첩 조건문을 가드 절로 변환
   - **설명 변수 도입**: 복잡한 표현식에 이름 부여
5. 린트를 수정한다: `./gradlew ktlintFormat`
6. 전체 테스트를 재확인한다: `./gradlew test`
   - 테스트 실패 시 변경을 되돌리고 보고한다
7. 결과를 보고한다:
   - 수행한 구조적 변경 목록
   - 변경된 파일 목록
   - 테스트 결과

## 규칙

- **동작 변경 절대 금지** — 외부에서 관찰 가능한 동작은 변하지 않아야 한다
- 테스트가 모두 통과하는 상태에서만 수행
- 한 번에 하나의 구조적 변경만 수행
- 변경 후 테스트가 실패하면 즉시 되돌린다
- 이 커맨드로 수행한 변경은 반드시 `[구조]` 유형으로 커밋한다
- 행위적 변경이 필요하면 개발자에게 별도 작업을 제안한다
