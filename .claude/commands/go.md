plan.md를 읽고 다음 미완료(체크되지 않은) 테스트를 찾아 TDD 사이클을 수행한다.

## 절차

1. `plan.md`를 읽는다
   - 파일이 없으면 "plan.md가 없습니다. `/plan` 커맨드로 먼저 작성해주세요."라고 안내하고 중단한다
2. 진행 상황을 요약한다: `완료: N개 / 미완료: M개 / 전체: N+M개`
3. `- [ ]`로 표시된 다음 미완료 항목을 찾는다
   - 미완료 항목이 없으면 "모든 항목이 완료되었습니다."라고 보고하고 중단한다
4. 선택한 항목을 개발자에게 알린다: "다음 항목을 진행합니다: {항목 내용}"
5. **Red**: 해당 항목의 실패하는 테스트를 작성한다
   - 3A 원칙: Arrange(준비) → Act(실행) → Assert(검증)
   - `@Nested` + `@DisplayName`(한국어) BDD 스타일
6. 테스트가 실패하는지 확인한다:
   ```
   ./gradlew :apps:commerce-api:test --tests "해당테스트클래스.해당테스트메서드"
   ```
7. **Green**: 테스트를 통과시키기 위한 최소한의 코드만 구현한다
   - 오버엔지니어링 금지
   - 기존 프로젝트 패턴 준수
8. 전체 테스트를 실행해 통과를 확인한다:
   ```
   ./gradlew test
   ```
9. 전체 테스트가 통과하면 `plan.md`에서 완료한 항목을 `- [x]`로 체크한다
10. 결과를 보고한다:
    - 진행한 항목
    - 작성한 테스트명
    - 구현한 코드 파일 목록
    - 전체 테스트 결과
    - 남은 미완료 항목 수

## 주의사항

- 실제 동작하지 않는 코드, 불필요한 Mock 구현 금지
- null-safety 준수
- println 금지
- 기존 코드 패턴과 일관성 유지
- 한 번에 하나의 테스트만 진행
- 전체 테스트가 실패하면 plan.md를 체크하지 않고 실패 내용을 보고한다
