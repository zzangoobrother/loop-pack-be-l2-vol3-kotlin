결함 수정 TDD: 버그를 재현하는 테스트를 먼저 작성하고, 수정한다.

결함 설명: $ARGUMENTS

## 절차

1. 결함을 분석한다: `$ARGUMENTS`에서 증상, 재현 조건, 기대 동작을 파악한다
2. **Step 1 — API 레벨 실패 테스트 작성**
   - E2E 또는 통합 테스트로 결함을 재현하는 테스트를 작성한다
   - 테스트가 실패하는지 확인한다 (결함이 재현되는지 확인)
3. **Step 2 — 최소 재현 테스트 작성**
   - 결함의 근본 원인을 좁히는 단위 테스트를 작성한다
   - 가능한 가장 작은 범위에서 결함을 재현한다
4. **Step 3 — 최소한의 수정**
   - 두 테스트를 모두 통과시키는 최소한의 코드를 작성한다
   - 관련 없는 코드는 변경하지 않는다
5. 전체 테스트를 실행해 사이드 이펙트가 없는지 확인한다:
   ```
   ./gradlew test
   ```
6. 결과를 보고한다:
   - 결함 원인 분석
   - 재현 테스트 목록
   - 수정 내용 요약
   - 전체 테스트 결과

## 규칙

- 반드시 실패 테스트를 먼저 작성한다 (테스트 없이 수정 금지)
- API 레벨 테스트 + 최소 재현 테스트 두 가지를 모두 작성한다
- 수정은 결함 해결에 필요한 최소한의 범위로 한정한다
- null-safety 준수
- 수정 후 기존 테스트가 깨지지 않는지 반드시 확인한다
