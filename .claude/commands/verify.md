커밋 전 최종 검증: 린트 + 전체 테스트를 실행하고 결과를 보고한다.

## 절차

1. 린트 체크를 실행한다: `./gradlew ktlintCheck`
2. 전체 테스트를 실행한다: `./gradlew test`
3. 변경된 파일 목록을 확인한다: `git status`
4. 결과를 아래 형식으로 보고한다:

### 보고 형식

```
## 검증 결과

### 린트 (ktlintCheck)
- 상태: ✅ 통과 / ❌ 실패
- 실패 시: 파일명, 위치, 위반 규칙

### 테스트
- 상태: ✅ 통과 / ❌ 실패
- 전체: N개 / 성공: N개 / 실패: N개
- 실패 시: 테스트명, 실패 사유

### 변경 파일
- 새 파일: N개
- 수정 파일: N개
- 삭제 파일: N개

### 판정
- ✅ 커밋 가능 → `/commit`으로 진행하세요
- ❌ 수정 필요 → 아래 항목을 수정하세요
  - (수정 필요 항목 목록)
```

## 규칙

- 린트 실패 시 `./gradlew ktlintFormat`으로 자동 수정 가능함을 안내한다
- 테스트 실패 시 실패한 테스트의 상세 내용을 포함한다
- 판정은 린트와 테스트 모두 통과해야 "커밋 가능"이다
